My Research

My research project is to create an interactive map using the Twitter data and a NLP emotion classifier that could potentially be a useful tool for analyzing Chicago gang activity.

The Twitter data used for the project is over 2 million tweets collected from members of a small gang in Chicago.

The emotion classifier I use on my data is one created for the specific vernacular used by the gang members. It was developed by Terra Blevins.

I will gather the data by parsing the tweets to filter the streets, blocks, and gang names mentioned in handles, tweet content, and retweets in the Chicago gang dataset. I will use a dictionary to cross reference blocks and street names in the area to be searched, which will be manually entered by the user of the program (i.e. 62, 39, Vernon). I will also create a dictionary of the known gangs in the area, and these will be searched for too. My program will create a new dictionary to associate each user with every place and gang mentioned. If a user retweets someone else’s tweet that contains a location, the location will be associated with the original tweeter. Then a “pin” representation of each tweet would be created, containing the location, handle, content, date created, and aggression/loss score generated by Terra’s emotion classifier. Potentially I could triangulate or average the locations to figure out where the user spends the most time.

I will plot the pins or triangulations on an interactive map. The pins will be the color of the aggression/loss score (i.e. red for aggressive, blue for loss on a gradient scale). The map will be adjustable so that users can filter tweets by date and key phrase. Dates can also be filtered so the user could search for a specific event such as a death or birthday (i.e. “RIP Tyquan” or “HBD”). This would allow us insight into where people are talking about events that happen in the community, on a somewhat fine-grained scale.

On the same map, I will plot neighborhood blight and arrests. The map will allow the user to toggle this data on and off. The user will be able to filter arrests by type (i.e. substance abuse, homicide). I will then see if my fellow researcher, Kevin Li of the statistics department, could potentially help me find some sort of correlation between the tweets and any of these variables.

This tool would be specific to this dataset, as the tweets come from a very specific set of people and the vernacular of each gang in each city varies vastly. But, if given different a different set of tweets (with an adjusted NLP classifier), a new dictionary of relevant blocks/streets/gangs, and police data from the area, it could hopefully be applicable to more than one region or city.

My goal for the project is to get an understanding of how computer science is used in academic research. I would love to create a useful and relevant tool, but I value the experience and learning process over any results I might find. However, it would be incredible if my final project is an accurate representation of my findings that is easy to use and robust.

I spent a lot of time parsing the data and organizing my python stripping code. I had to consider a lot of different tweet formats, such as retweets, emojis, and http links. It is requiring me to refresh the Python skills I learned 2 years ago and get back to the basics.

My parser worked well on a tiny dataset of 5 tweets, successfully pulling out gang and location mentions, and associating information with the correct user (i.e. if its a retweet).
